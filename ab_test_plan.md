# A/B Test Plan

# Цель теста
Сравнить текущую версию модели (`v1`) и новую версию (`v2`) и определить, улучшает ли новая модель качество выявления клиентов с высоким риском дефолта.

# Гипотеза
Модель `v2` лучше выявляет клиентов с риском дефолта и позволяет снизить финансовые потери банка по сравнению с `v1`.

# Группы
- Группа A—текущая модель `v1`
- Группа B—новая модель `v2`

# Способ разделения трафика
Трафик делится случайным образом:
- 50% запросов направляются на `v1`
- 50% запросов направляются на `v2`

# Продолжительность теста
Тест продолжается до накопления статистически достаточного объема наблюдений.

# Основная метрика
- F1-score для класса дефолта

# Дополнительная метрика
- Recall для класса дефолта

# Почему выбраны эти метрики
F1-score важен при несбалансированных классах, потому что учитывает и precision, и recall.
Recall особенно важен в задаче кредитного скоринга, потому что помогает не пропускать клиентов с высоким риском дефолта.

# Бизнес-метрики
- снижение доли клиентов, ушедших в дефолт после одобрения;
- снижение финансовых потерь от невозвратов;
- улучшение качества отбора рискованных клиентов.

# Статистический анализ
Для сравнения результатов можно использовать:
- z-test для пропорций;
- доверительные интервалы для оценки различий между вариантами.

# Критерий успеха
Модель `v2` считается успешной, если:
- улучшает F1-score;
- не ухудшает критично recall;
- показывает статистически значимое улучшение;
- дает положительный бизнес-эффект.

# Риски
- недостаточный размер выборки;
- влияние внешних факторов;
- смещение трафика между группами;
- преждевременное завершение теста.

# Итоговое решение
По завершении теста принимается решение:
- оставить `v1`;
- перевести систему на `v2`;
- доработать `v2` и повторить эксперимент.
